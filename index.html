Robot Identity &
Memory Subnet

MemoryMesh AI gives every physical robot a persistent on-chain identity, verifiable task memory, and a live reputation score — powered by Proof-of-Physical-Work.

Core Primitives
🆔
RobotID
Cryptographic on-chain identity derived from hardware fingerprint + owner address. Immutable. Portable across all Konnex subnet workloads.
🧠
MemoryStore
Append-only PoPW-verified task log per robot. Miners read memory history to generate better policies. Every mission improves the next.
⭐
ReputationScore
Weighted on-chain score updated after every verified PoPW event. Enterprises query reputation before hiring robots autonomously.
⛏️
Memory-Enhanced Mining
Miners query a robot's MemoryStore to select optimal execution strategies. Historical success patterns directly improve PoPW scores.
✅
PoPW Validation
Validators score real-world robot execution across safety, task match, and efficiency dimensions. Scores are written on-chain permanently.
🔗
Cross-Subnet Utility
RobotID and MemoryStore can be consumed by Drone, RoboArm, and SLAM subnets — creating compounding network value across Konnex.
How It Works
📋
1. Task Posted
Instruction hashed on-chain with RobotID attached
🧠
2. Memory Query
Miner reads robot's history to generate optimal policy
🤖
3. Execution
Robot executes policy — sensor data captured
⚖️
4. PoPW Score
Validator scores safety, match & efficiency
💾
5. Memory Write
Outcome appended — reputation updated on-chain
💰
6. Settlement
testKNX rewards flow to miners & validators
